---
name: Multi-Provider Support
description: ai-core should evolve from Gemini-first runtime primitives into provider-aware, model-aware, auth-aware, routing-capable infrastructure inspired by opencode
type: feedback
---

`opencode` demonstrates that solving provider flexibility requires more than adding more keys.

ai-core should evolve toward:
- provider schema
- model capability registry
- provider adapters
- auth abstraction
- routing policy

Gemini remains the default first-class provider, but ai-core should stop assuming Gemini is the only provider forever.
